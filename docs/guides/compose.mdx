The Tailcall CLI's compose command enables the conversion of configuration files between three supported formats: gql or graphql, yml or yaml, and json, with json set as the default format.

## Converting Formats

To convert files between different formats, use the following syntax:

```bash
tailcall compose <input_files> <output_file> --format format_type
```

## Example Usage

Let's try to convert a Tailcall graphql file to json and then back to graphql

To convert graphql to json

```bash
tailcall compose examples/jsonplaceholder.graphql --format json > "examples/jsonplaceholder.json"
```

Now to convert back to graphql

```bash
tailcall compose examples/jsonplaceholder.json --format graphql > "examples/jsonplaceholder2.graphql"
```

To learn more about writing configuration to leverage the full power of Tailcall, explore the [Operators](../operators/index.md) documentation.

<TabItem value="graphql" label="graphql">
`schema.json`

```graphql showLineNumbers
schema
  @server(port: 8000, graphiql: true, hostname: "0.0.0.0")
  @upstream(baseURL: "http://jsonplaceholder.typicode.com", httpCache: true, batch: {delay: 100}) {
  query: Query
}

type Query {
  posts: [Post] @http(path: "/posts")
  users: [User] @http(path: "/users")
  user(id: Int!): User @http(path: "/users/{{args.id}}")
}

type User {
  id: Int!
  name: String!
  username: String!
  email: String!
  phone: String
  website: String
}

type Post {
  id: Int!
  userId: Int!
  title: String!
  body: String!
  user: User @call(query: "user", args: {id: "{{value.userId}}"})
}
```
